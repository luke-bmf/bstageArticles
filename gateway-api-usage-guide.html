b.stage Gateway API 개발자 가이드

본 가이드는 b.stage의 커스텀 빌드(Custom Build) 기능을 사용하여 독립적인 프런트엔드 환경이나 외부 서비스를 구축하려는 개발자를 대상으로 합니다. Gateway API를 통해 b.stage의 백엔드 인프라를 그대로 활용하면서, UI/UX는 자유롭게 커스터마이징할 수 있습니다.

1. 개요

b.stage Gateway API는 SaaS 플랫폼의 데이터와 비즈니스 로직을 외부로 노출하는 인터페이스입니다. 개발자는 이 API를 호출하여 알림, 투표, 콘텐츠 랭킹 등 b.stage의 핵심 기능을 자신의 서비스에 통합할 수 있습니다.

be-qa.static.astage.in https://be-qa.static.astage.in/gateway-api-docs/public/index.html

2. 인증 및 보안 (Authentication)

Gateway API에 접근하기 위해서는 모든 HTTP 요청 헤더에 아래의 인증 키를 포함해야 합니다.

Header Name Type Description
X-BSTAGE-APP-ID String b.stage 관리자 센터에서 발급받은 애플리케이션 ID
X-BSTAGE-APP-SECRET String 보안 인증을 위한 비밀 키
X-BSTAGE-TENANT-ID String 서비스를 식별하는 테넌트 ID
authorization String (선택) 로그인 기반 API 호출 시 필요한 유저 인증 토큰

3. 주요 API 서비스 그룹

서비스 그룹 주요 기능 및 특징 활용 사례
Home API • 서비스의 메인 레이아웃 및 구성 데이터 제공
• 테넌트별 맞춤 홈 화면 구성을 위한 기반 데이터 호출		 브랜드 아이덴티티가 반영된 커스텀 홈 화면 구축
Content API • 플랫폼 내 다양한 콘텐츠 데이터 제공
• 게시글 및 미디어 콘텐츠 정보 연동		 독창적인 레이아웃의 콘텐츠 상세 페이지 및 리스트 구현
Notification V2 • 푸시 토큰 등록/삭제 및 알림 설정 관리
• 최근 알림 기록 및 페이지네이션 기반 목록 조회		 실시간 유저 인터랙션 및 개인화된 알림 시스템 구축
Ranking Poll • 투표권 잔액 조회 및 투표 참여 로직 제공
• 후보자 목록 및 실시간 득표 순위/통계 조회		 오디션, 어워즈 등 참여형 이벤트 페이지 구현
Recommend (Data) • 반응도 기반 게시판 콘텐츠 랭킹 스냅샷 제공
• 데이터 분석 기반의 인기 게시글 리스트 반환		 메인 페이지 인기 섹션 및 추천 피드 구성

4. 구현 단계 (Implementation Steps)

  1. 프로젝트 설정: b.stage 관리자 설정에서 커스텀 빌드 권한을 활성화하고 API 키 세트를 발급받습니다.
  2. 기본 환경 구축: API Base URL(https://gateway-public.qa.bstage.systems)을 설정하고 공통 헤더를 구성합니다.
  3. API 호출: 목적에 맞는 API(예: /api/notification/v2/...)를 호출하여 데이터를 연동합니다.
  4. 응답 처리: JSON 형식으로 반환되는 데이터를 파싱하여 커스텀 UI를 렌더링합니다.
